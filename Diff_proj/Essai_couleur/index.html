L'égalité == vs ===

Il y a deux façons de vérifier si deux valeurs sont égales en JavaScript : == et ===, aussi appelées égalité simple et égalité stricte :
L'égalité simple "==" vérifie la valeur, mais pas le type ! => 5 == "5" => TRUE

L'égalité stricte "===" vérifie à la fois la valeur et le type => 5 === "5" => FALSE
Etant donné que ici, nous comparant un number à un string

Nous avons exactement les mêmes relations entre "!=" et "!=="

Les types d'erreurs

  • Erreurs de synthaxe : erreurs dans l'utilisation de JavaScript : fautes d'orthographe, crochets manquants, nombre de guillemets incorrect, etc. ;
  • Erreurs de logique : Problèmes de logique dans le programme. Ce type d'erreur ne seront pas affiché par la console ou le terminal et peuvent conduire à des boucles infinie.
    • affectation de valeur erronnée à une variable.
    • mélanger des conditions dans une boucle "if"
    • mauvais ordre dans l'écriture des lignes ou des blocs de code.
  • Erreurs d'éxecution : Mauvaise exécution lié à un aspect externe à l'application. Ce type d'erreur est commune et doit être prise en compte dans le code.
    exemple ligne 568 du code JS, avec une boucle if ou une fonction "try" ou "catch"
